Технічна специфікація мови JDS

Цей документ містить опис архітектури мови JDS (JSON-based/Just-asm Data Script) та структуру її документації для забезпечення цілісності майбутніх правок.


1. Загальні відомості

Цільова платформа: x86_64 Linux (ASM NASM).

Тип: Статично структурована скриптова мова з прямою компіляцією в машинний код.

Тип даних: Єдиний тип — 64-бітне ціле число (Signed Integer).

2. Синтаксис мови (AST вузли)

Змінні та Арифметика

let <name> [= <expr>]: Оголошення змінної.

Оператори: +, -, *, /.

Пріоритет: Множення та ділення мають вищий пріоритет (реалізовано через parse_multiplicative).

Керування потоком

if, elif, else: Умовні переходи.

while: Цикли.

Оператори порівняння: ==, <, >.

Системні команди

print <string|term>: Вивід у STDOUT. Підтримує екранування (\n, \t).

input <var>: Зчитування цілого числа з STDIN.

exit <term>: Системний виклик 60 (exit) з кодом повернення.

argc: Вбудована змінна (кількість аргументів командного рядка).

3. Модульність та Динамічне виконання

Ключове слово module: Якщо на початку файлу, компілюється в сирий бінарний код (-f bin) без заголовка _start.

Команда exec "<path>":

Відкриває файл.

Виділяє пам'ять через mmap (RWX).

Завантажує вміст.

Викликає код як функцію (CALL).

4. Специфікація Документації (HTML/CSS)

Layout: Фіксований сайдбар зліва (260px) + основний контент з max-width: 850px, центрований по горизонталі (justify-content: center).

Стиль: Темна тема (#0f172a), акцентний колір — блакитний (#38bdf8).

Локалізація: Перемикач мов (EN/UK) через JavaScript, де контент розділений за класами .lang-section.

Код: Блоки <pre> з атрибутом white-space: pre для збереження форматування.

5. Компіляція (Rust Backend)

Lexer: Обробляє токени, ігнорує коментарі //.

Parser: Рекурсивний спуск (Recursive Descent).

Compiler: Генерація x86_64 ASM з використанням стандартної бібліотеки (функції _print_rax, _read_int, _load_and_run).
